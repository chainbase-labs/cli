# Tops CLI Command Design

## Overview

Add a `tops` command group to expose [Tops](https://tops.chainbase.com) crypto social intelligence APIs. No API key required — endpoints are public but rate limited per IP (10 req/s, 60 req/min, 600 req/hour).

## Changes

### 1. New command group: `tops`

File: `src/commands/tops.ts`

Subcommands:

| Subcommand | API Path | Description |
|---|---|---|
| `trending` | `GET /v1/tool/list-trending-topics` | List trending crypto topics |
| `topic <topic_id>` | `GET /v1/tool/get-topic` | Structured details for a topic |
| `posts <topic_id>` | `GET /v1/tool/get-topic-posts` | Twitter/X posts for a topic |
| `mentions <keyword>` | `GET /v1/tool/search-mentions` | Search recent X mentions by keyword |
| `search <keyword>` | `GET /v1/tool/search-narrative-candidates` | Search narrative candidate topics |

### 2. HTTP client

Uses `axios` directly (not `ChainbaseClient`) with base URL `https://api.chainbase.com/tops`. No `x-api-key` header needed.

### 3. Options

- `trending` supports `--language <lang>` (zh/en/ko, default: en)
- All other subcommands inherit global `--json` option from parent

### 4. Registration

Add `program.addCommand(createTopsCommand())` in `src/index.ts`.

### 5. README

Document the new command group under a `### tops` section with usage examples.
